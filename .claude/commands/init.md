# Команда /init — инициализация сессии с полным контекстом

Выполни последовательно три вызова MCP-инструментов `agent-context`:

1. `start_session` с `project_path = "E:/Scout"` — открыть сессию, получить
   контекст проекта и данные предыдущей сессии
2. `server_status` — получить текущие метрики сервера, статусы контейнеров,
   события за последний час
3. `get_context` — получить полный контекст включая чекпоинты активной сессии

После всех трёх вызовов — выдай краткую сводку:
- Номер открытой сессии
- Состояние сервера (CPU, RAM, инциденты)
- Статус контейнера `scout-mcp-1` (running / stopped / отсутствует)
- Что было в предыдущей сессии (если есть данные)
- Задачи в backlog (список файлов из `E:\Scout\Tasks\backlog\`)
