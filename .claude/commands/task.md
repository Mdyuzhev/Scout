# Команда /task — взять задачу в работу

## Назначение

Команда `/task` запускает рабочий цикл: читает контекст, выбирает задачу из
backlog, выполняет её, обновляет `CLAUDE.md` и переносит задачу в `done/`.

---

## Структура директорий задач

```
E:\Scout\Tasks\
├── backlog\          # задачи ожидающие выполнения (SC-NNN_slug.md)
└── done\             # выполненные задачи (с добавленным разделом ## ✅)
```

---

## Алгоритм выполнения /task

### Шаг 1 — Актуализация контекста

Прочитать `.claude/CLAUDE.md` — архитектура, стек, сервисы, порты, ограничения.
Если в CLAUDE.md есть раздел `⚠️ Известные проблемы` — учесть перед началом.

### Шаг 2 — Выбор задачи

Прочитать список файлов в `Tasks/backlog/`.

- **Backlog пуст** — сообщить: "Backlog пуст. Добавь задачу в `Tasks/backlog/`."
- **Одна задача** — взять автоматически, сообщить: "Беру в работу: SC-XXX — название"
- **Несколько задач** — показать нумерованный список, спросить какую взять,
  дождаться ответа пользователя.

### Шаг 3 — Выполнение задачи

Прочитать файл задачи. Выполнять строго по порядку шагов из файла.
Подключение к серверу — только через paramiko (шаблон в CLAUDE.md).

Если обнаружена новая проблема вне рамок задачи — зафиксировать в разделе
"Обнаруженные проблемы" и продолжать. Не останавливаться.

### Шаг 4 — Завершение задачи

**4a. Обновить CLAUDE.md:**
- Статусы сервисов если изменились
- Раздел `⚠️ Известные проблемы` — добавить/убрать записи
- Структуру проекта если добавились файлы
- Реестр задач: перевести SC-NNN из `🔲 в очереди` в `✅ выполнена`
- Дату `*Последнее обновление*`

**4b. Добавить раздел в файл задачи:**

```markdown
---

## ✅ Статус: ВЫПОЛНЕНА

**Дата завершения:** YYYY-MM-DD

**Что сделано:**
- (3-7 пунктов конкретных изменений)

**Отклонения от плана:**
- (если были; иначе — "Выполнено в соответствии с планом")

**Обнаруженные проблемы (перешли в следующие задачи):**
- (если есть; если нет — убрать этот пункт)
```

**4c. Переместить файл:**
`Tasks/backlog/SC-NNN_slug.md` → `Tasks/done/SC-NNN_slug.md`

### Шаг 5 — Отчёт пользователю

```
## ✅ Задача SC-NNN выполнена

**Задача:** (название)
**Время:** (примерное)

### Что сделано
(3-7 пунктов с конкретными изменениями)

### Проверка критериев
(критерий → результат)

### Изменения в CLAUDE.md
(что обновлено)

### Следующие шаги
(если есть незакрытые вопросы или задачи которые стоит создать)
```

---

## Важные правила

Никогда не начинать выполнение не прочитав CLAUDE.md.

Баги и улучшения за рамками текущей задачи — не делать сейчас. Зафиксировать как
"Обнаруженные проблемы" и предложить создать новую задачу.

Отчёт — всегда, даже если задача простая.
