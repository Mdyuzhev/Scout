# Команда /research — запустить полный цикл исследования

## Назначение

Запускает полный исследовательский пайплайн Scout в фоне.
Агент сам собирает URL (web_search, подписка), передаёт их Scout для индексации,
затем получает чанки и генерирует бриф своим LLM (подписка).

SC-036: Scout = детерминированный инструмент, LLM на стороне агента.

---

## Алгоритм выполнения

### Шаг 1 — Проверить сервер

Вызвать `mcp__homelab__run_shell_command` с командой:
```
curl -s http://localhost:8020/health
```

Если не `{"status":"ok"}` — сообщить пользователю и остановиться.

### Шаг 2 — Получить входные данные от пользователя

Спросить три вещи:

**1. Тема** — что исследуем. Пример: `"рынок труда России 2024-2025"`

**2. Запрос для брифа** — что синтезировать. Пример: `"дефицит кадров по отраслям, динамика зарплат, влияние ВПК"`

**3. Модель** (по умолчанию `opus`): `haiku` / `sonnet` / `opus`

### Шаг 3 — Собрать URL (агент, подписка)

Агент выполняет web_search по теме. Целевое количество: 150-200 URL.
Составить 4-6 поисковых запросов (статистика, аналитика, тренды, прогнозы).
Объединить результаты, дедуплицировать.

Сохранить URL в `E:\Scout\results\urls\YYYY-MM-DD_<slug>_urls.txt`.

### Шаг 4 — Создать job и передать URL

```
scout_create_job(
    topic: <тема>,
    query: <запрос>,
    model: <модель>,
    language: "ru",
    top_k: 15,
    save_to: "/opt/scout/results/YYYY-MM-DD_<slug>.md"
)
→ job_id
```

Передать URL батчами:
```
scout_push_urls(job_id, urls[:100], batch_label="batch_1")
scout_push_urls(job_id, urls[100:], batch_label="batch_2", is_final=True)
```

### Шаг 5 — Ждать индексации

Опрашивать `scout_job_status(job_id)` каждые 3 минуты.
Максимум 6 проверок (18 минут). Ждать `stage=indexing_done`.

При каждой проверке писать пользователю:
```
⏳ [N мин] stage=<stage> — <message>
```

### Шаг 6 — Получить чанки и сгенерировать бриф

```
ctx = scout_get_context(session_id, query, top_k=15)
```

Агент генерирует бриф из `ctx["context"]` своим LLM (подписка).
Использовать системный промпт для структуры брифа.

### Шаг 7 — Сохранить бриф

```
scout_save_brief(session_id, brief, model="claude-opus-4-6", save_to="/opt/scout/results/...")
```

Сохранить локально в `E:\Scout\results\<имя файла>`.

### Шаг 8 — Отчёт пользователю

```
## ✅ Исследование завершено

**Тема**: <тема>
**Модель**: <model>

### Источники
- URL собрано: <кол-во>
- Проиндексировано: <documents_count> документов
- Чанков: <chunks_count> | В брифе: <sources_used>

### Бриф сохранён
- Сервер: <saved_to>
- Локально: E:\Scout\results\<файл>

### Session ID для уточняющего поиска
`<session_id>`
```

---

## Правила

Никогда не использовать nohup или shell-скрипты — только MCP-инструменты.
Если `status == "failed"` — показать `error` и предложить повторить с меньшим кол-вом URL.
Имя файла брифа: `YYYY-MM-DD_<slug>.md` — не перезаписывать предыдущие.
Slug: только латиница/цифры/дефис, max 40 символов.
НЕ использовать auto_collect — deprecated (SC-036).
