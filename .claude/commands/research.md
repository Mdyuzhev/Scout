# Команда /research — запустить полный цикл исследования

## Назначение

Запускает полный исследовательский пайплайн Scout в фоне:
Haiku автоматически ищет URL → Scout индексирует → модель генерирует бриф.
Агент вызывает `scout_research_async(auto_collect=True)` и опрашивает статус
через `scout_job_status` каждые 3 минуты. Никаких shell-скриптов.

---

## Алгоритм выполнения

### Шаг 1 — Проверить сервер

Вызвать `mcp__homelab__run_shell_command` с командой:
```
curl -s http://localhost:8020/health
```

Если не `{"status":"ok"}` — сообщить пользователю и остановиться.

### Шаг 2 — Получить входные данные от пользователя

Спросить три вещи:

**1. Тема** — что исследуем. Пример: `"рынок труда России 2024-2025"`

**2. Запрос для брифа** — что синтезировать. Пример: `"дефицит кадров по отраслям, динамика зарплат, влияние ВПК"`

**3. Модель** (по умолчанию `haiku`): `haiku` / `sonnet` / `opus`

### Шаг 3 — Запустить исследование через scout_research_async

Вызвать MCP-инструмент `scout_research_async` с параметрами:
```
topic:              <тема>
query:              <запрос>
auto_collect:       True
auto_collect_count: 150
model:              <модель>
language:           "ru"
top_k:              10
save_to:            "/opt/scout/results/YYYY-MM-DD_<slug>.md"
```

Сразу написать пользователю:

```
🚀 Исследование запущено (job_id: XXX)
Тема: <тема>
Модель: <модель>

Пайплайн работает в фоне (~5-15 мин):
  1. Haiku ищет ~150 URL через web_search (~1-2 мин)
  2. Scout тестирует 10 URL, затем индексирует все (~3-10 мин)
  3. <модель> генерирует бриф (~1 мин)

Проверяю статус каждые 3 минуты...
```

### Шаг 4 — Опрашивать статус каждые 3 минуты

Повторять вызов `scout_job_status(job_id=<job_id>)` каждые 3 минуты.
Максимум 6 проверок (18 минут), после — сообщить об истечении времени.

При каждой проверке писать пользователю:

```
⏳ [N мин] stage=<stage> — <message>
```

Ждать пока `status` не станет `"completed"` или `"failed"`.

### Шаг 5 — Получить результат

Когда `status == "completed"` — вызвать `scout_job_result(job_id=<job_id>)`.

Прочитать бриф с сервера:
```
mcp__homelab__run_shell_command: cat <saved_to>
```

Сохранить локально через Write/Edit в `E:\Scout\results\<имя файла>`.

### Шаг 6 — Отчёт пользователю

```
## ✅ Исследование завершено

**Тема**: <тема>
**Модель**: <model> | **Токены**: <tokens>

### Источники
- Haiku нашёл: <auto_collected> URL автоматически
- Проиндексировано: <documents_count> документов (<failed_count> failed)
- Чанков: <chunks_count> | Источников в брифе: <sources_used>

### Бриф сохранён
- Сервер: <saved_to>
- Локально: E:\Scout\results\<файл>

### Session ID для уточняющего поиска
`<session_id>`
```

---

## Правила

Никогда не использовать nohup или shell-скрипты — только MCP-инструменты.
Если `status == "failed"` — показать `error` и предложить повторить с меньшим `auto_collect_count`.
Если за 18 минут статус не сменился — вызвать `scout_job_status` ещё раз и сообщить пользователю.
Имя файла брифа: `YYYY-MM-DD_<slug>.md` — не перезаписывать предыдущие.
Slug: только латиница/цифры/дефис, max 40 символов.
